# 凛晶 / Inori 冰晶主题模板

[English](README.md) | [简体中文](README.zh-CN.md)

此目录是 `codex-desktop-theme-builder` Skill 内置的完整实现示例，不是仓库主体。

模板展示淡蓝冰晶主题：首页专属角色背景、星空信号画布、明显区分的任务进行中/已完成状态、隔离启动与停止脚本、快捷方式创建器，以及可选宠物包。

## 作为模板使用

1. 将整个目录复制到独立项目。
2. 若要派生其他主题，请替换美术素材，并重命名示例专用的 ID、class、标签、文件名、profile 名和包名。
3. 使用任何内置选择器前，先检查当前 Codex 版本的真实 DOM。
4. 保持运行路径相对于复制后的项目目录。
5. 验证首页、任务进行中、任务已完成、任务切换、菜单、减少动画、窄窗口和宠物透明状态。
6. 暂存或发布前运行 Skill 隐私审计。

不要直接修改或运行已安装 Skill 内的模板副本。

## 示例控制脚本

在 Windows 上运行 `start-inori-theme.cmd` 启动隔离主题实例，运行 `stop-inori-theme.cmd` 停止实例。启动器会在运行时查找已安装的 Codex，并把主题状态保存在模板目录之外。

可选辅助脚本：

- `create-inori-shortcut.cmd`：使用已安装 Codex 应用图标创建 Windows 快捷方式。
- `install-inori-pet.cmd`：独立安装示例宠物包。

这些脚本是需要复制、重命名和检查的示例，不是固定的公共 API。

## 架构说明

- `theme.css`：视觉 token 覆盖与任务状态样式。
- `injector.mjs`：管理带版本的样式、任务状态标记、装饰信号图和宠物窗口清理。
- `assets/inori-original-crystal-hero.png`：原创角色“凛晶 / Inori”的首页背景。
- `pet/inori-pet/`：可选宠物 manifest 与图集。

注入器绝不能给 `avatar-overlay` 目标应用主题。装饰层使用 `pointer-events: none`，重复动画在 `prefers-reduced-motion: reduce` 下关闭。

## 原创角色规范

“凛晶 / Inori”是为本模板设计的原创角色。首页主视觉与宠物必须保持同一识别系统：银蓝长发、冰蓝眼、居中切面晶冠、白色与冰蓝色不对称层叠晶装，以及胸前雪晶核心。扩展示例时不要重新引入粉发、红瞳、红黑舞台服或侧边发饰。

## 素材说明

位图素材由 AI 工具生成，角色“凛晶 / Inori”是为本模板设计的原创银蓝冰晶角色；本包中的“Inori”仅指该原创角色。

角色设计有意避开粉发、红瞳、红黑舞台服、侧边发饰等既有虚构角色的显著组合。“AI 生成”本身不能保证素材一定具备可版权性或排他性，因此后续许可证应明确是否覆盖这些素材。

参见[仓库许可证与素材边界](../../../../../README.zh-CN.md#许可证与素材边界)。

本模板不包含真实 Codex UI 截图。
